# Документация проекта "Анализ конкурентов"

## Структура проекта

- `app/config.py` — настройки приложения.
- `app/history.py` — работа с историей запросов.
- `app/parser.py` — парсер сайтов на Selenium (headless Chrome).
- `app/openai_client.py` — взаимодействие с OpenAI.
- `app/main.py` — FastAPI-приложение и маршруты.
- `static/index.html` — веб-интерфейс.
- `data/Links.txt` — список ссылок для анализа.
- `history.json` — автоматически создаётся при первом запросе.

## API

### `POST /analyze_text`

- Описание: анализ текстового описания конкурента.
- Параметры: `text` (json)
- Ответ:
  - `result` — строка с ответом от OpenAI.

Пример запроса:

```bash
curl -X POST http://127.0.0.1:8000/analyze_text \
  -H "Content-Type: application/json" \
  -d '{"text":"Описание конкурента"}'
```

### `POST /analyze_image`

- Описание: анализ изображения и его короткое описание.
- Параметры: `description` (form-data), `image` (file)
- Ответ:
  - `result` — строка с ответом от OpenAI.

Пример запроса:

```bash
curl -X POST http://127.0.0.1:8000/analyze_image \
  -F "description=Сайт конкурента" \
  -F "image=@banner.jpg"
```

### `POST /parse_demo`

- Описание: парсинг сайта по URL и анализ извлечённых данных.
- Параметры: `url` (json)
- Ответ:
  - `parsed` — объект с `title`, `h1`, `first_paragraph`
  - `result` — ответ OpenAI.

Пример запроса:

```bash
curl -X POST http://127.0.0.1:8000/parse_demo \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com"}'
```

### `GET /history`

- Описание: возвращает последние 10 запросов.
- Ответ:
  - `history` — список записей.

Пример запроса:

```bash
curl http://127.0.0.1:8000/history
```

### `GET /links`

- Описание: возвращает список ссылок для анализа из файла `data/Links.txt`.
- Ответ:
  - `links` — список URL.

Пример запроса:

```bash
curl http://127.0.0.1:8000/links
```

### `POST /analyze_links`

- Описание: анализирует все ссылки из файла `data/Links.txt`.
- Ответ:
  - `links` — список URL.
  - `analysis` — массив объектов с результатами анализа для каждой ссылки.

Пример запроса:

```bash
curl -X POST http://127.0.0.1:8000/analyze_links
```

## Мультимодальные функции

- Текст: анализирует конкурентные тексты, определяет сильные и слабые стороны, уникальные предложения и рекомендации.
- Изображения: по описанию изображения создаёт маркетинговые инсайты и оценку визуального стиля.
- Веб-сбор данных: извлекает базовый контент сайта (`title`, `h1`, первый абзац) и делает выводы.
